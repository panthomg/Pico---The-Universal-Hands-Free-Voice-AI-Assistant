[ Concept Pitch ]

Why Pico Exists

Unlike typical smart speakers designed primarily as sales portals, Pico is built to teach and assist. By removing screens, distractions, and notifications, Pico acts as a stationary room-anchored assistant dedicated to high-fidelity logical exchange, creativity, and knowledge access.

// Target Audiences:

  • Students: An infinitely patient, factual mentor for learning concepts step-by-step.
  • Professionals: A multi-model research partner for brainstorming and development.
  • Elderly: A conversational, voice-only interface that avoids complex smartphone UIs.
  • Creators: A streamlined workspace helper for distraction-free idea generation.
🔓

Freedom of Knowledge

No ecosystem lock-in. Switch backend engines (DeepSeek, OpenAI, Claude, Gemini, or local models via Ollama) with a single click. Your API keys are stored locally and encrypted.

🎙️

360° Far-Field Capture

Equipped with a dual MEMS microphone array and an XMOS core, Pico captures voice commands up to 5 meters away, even during active audio playback.

🧠

Adaptive Modes

Utilizes three processing modes: Think (reasoning and code generation), Search (real-time data summaries), and Think & Search (deep analytical synthesis).

⏳

Smart Interrupt

Equipped with on-device Voice Activity Detection (Silero VAD). Say a command mid-sentence and the speaker halts immediately to parse your new request.

The Brain

Arduino UNO Q 4GB

The Ears

ReSpeaker Lite XVF3800

XMOS hardware layer far-field tracking allows clear, cross-room vocal commands under complex acoustic echo environments.

Proposed Project Title

Pico — For Those Who Ask

Pico — For Those Who Ask
Problem Being Solved
The world has unlimited information, yet knowledge is harder to access than ever. Voice assistants exist, but they are designed to sell products, not to teach. They give quick answers, not deep understanding. They lock you into one ecosystem, one AI provider, one voice, one personality. Students, professionals, the elderly, and the curious are left behind — unable to customize, unable to choose, and unable to truly learn. Pico solves this by being a voice assistant built for the curious mind — a companion that helps you understand, explore, and learn. It gives you freedom: choose any AI provider, customize everything, and never get locked in. No subscriptions. No barriers. Just knowledge, accessible to everyone.
Project Idea & Core Architecture
Pico is a hands-free voice AI assistant built on Arduino UNO Q. Just say "Hey Pico" and ask anything — it searches the web, thinks creatively, and synthesizes deep insights, all in a natural voice. Three Modes for Learning: Think: Pure creativity (poetry, stories, ideas); Search: Real-time information (news, facts, weather); Think & Search: Deep synthesis (research, analysis, understanding). Key Features include: Choose any AI provider (DeepSeek, OpenAI, Claude, Gemini, Groq, Ollama) — switch anytime; Choose your voice, personality, and wake word; Far-field 2-mic array captures voice from up to 3 meters; Complete web app for dashboard, settings, chat history, and remote mode; Smart home control (MQTT, Matter, WiZ lights). Built on Arduino UNO Q — affordable and open.
Board Selection and Integrated Role
The Arduino UNO Q (ABX00162) is the central processing unit of Pico. It runs Debian Linux and hosts our custom Pico software — a complete voice assistant framework built from scratch in Python. This software handles the entire audio pipeline: wake word detection (Picovoice Porcupine), speaker verification (Picovoice Eagle), offline speech-to-text (Vosk), natural text-to-speech (Piper TTS), and smart interrupt (Silero VAD). It connects to the ReSpeaker Lite via USB for far-field voice capture, processes voice commands through our intelligent command router (Think / Search / Think & Search modes), and handles AI API calls to DeepSeek, OpenAI, or any other provider the user chooses. In addition to the core voice engine, the UNO Q hosts a full-featured companion web app built with React and FastAPI. The UNO Q's dual-brain architecture (Linux MPU + STM32 MCU) ensures stable audio processing, real-time streaming via WebSockets, and seamless smart home control (MQTT, Matter, WiZ lights). This combination of custom Pico software and a complete web app ecosystem makes Pico a production-ready, commercial-grade voice assistant platform.
Software Stack, Toolchain & Testing Frameworks
We are building Pico using a modern, production-grade software stack. The Arduino UNO Q runs Debian Linux, and all application logic is written in Python 3.11+, making development fast and flexible. For the backend API and real-time communication, we are using FastAPI with Uvicorn as the ASGI server, which provides high-performance REST APIs and WebSocket support for live streaming. The companion web app is built with React and TypeScript for the frontend, styled with Tailwind CSS, and communicates with the backend via REST APIs and WebSockets. Core modules include Porcupine (wake word), Eagle (speaker ID), Vosk (local STT), Piper (local TTS), and Silero (VAD/barge-in). Developers use VS Code with the Remote SSH extension. Our testing plan includes: Unit Testing (individually parsing modules via unittest); Integration Testing (measuring complete audio paths via arecord/aplay and time modules); and Performance Testing (monitoring API costs and local CPU/RAM performance on the MPU).

def _box_mesh(x1, y1, z1, x2, y2, z2):
    xlo, xhi = sorted((x1, x2))
    ylo, yhi = sorted((y1, y2))
    zlo, zhi = sorted((z1, z2))
    extents = [xhi - xlo, yhi - ylo, zhi - zlo]
    center = [(xlo + xhi) / 2, (ylo + yhi) / 2, (zlo + zhi) / 2]
    box = trimesh.creation.box(extents=extents)
    box.apply_translation(center)
    return box
